זה הליבה של מה שעלה. הסדר חשוב — כל שלב בונה על הקודם.

### 2.0 הכנה חד-פעמית

1. ב-WhatsApp בטלפון שלך, **שמור איש קשר חדש** בשם "מקובצקי" עם המספר `+972539456995`.
2. בחר **קבוצה אמיתית של פרויקט** שתשמש כקבוצת בדיקה. הצעה: קבוצה קטנה / קבוצת בדיקה ייעודית, לא קבוצה רשמית של לקוח עד שתסיים את הבדיקה. אם אין — תיצור קבוצה חדשה עם עצמך בלבד.
3. **הוסף את "מקובצקי" כחבר בקבוצה**.

**תוצאה צפויה:** בקבוצה רואים שמספר חדש נוסף. ייתכן ש-Vercel logs ירשמו `incomingMessageReceived` מאיזושהי הודעת מערכת של "הצטרף לקבוצה" — לא קריטי, ייתכן שיהיה skipped.

### 2.1 קליטת orphan ראשון

**מטרה:** לראות שה-webhook מזהה קבוצה חדשה ומייצר רשומה מסוג orphan.

**שלבים:**
1. בקבוצה שיצרת/בחרת, שלח את ההודעה: `@מקובצקי בדיקה ראשונה`
   (השתמש ב-WhatsApp's @-mention UI — תקליד `@` ובחר את "מקובצקי" מהרשימה. אם מופיע אצלך כשם השמור — בחר אותו. אם לא, הקלד ידנית.)
2. המתן ~5-10 שניות.
3. פתח/רענן את https://makovetzky-erp.vercel.app/inbox

**תוצאה צפויה:**
- למעלה צצה סקציה צהובה: **"קבוצות WhatsApp ממתינות לקישור (1)"**.
- בשורה: שם הקבוצה, ה-chatId שלה (משהו כמו `972…-1638…@g.us`), זמן קבלה.
- ה-PendingDocument עצמו מופיע גם בטבלה הראשית של /inbox עם הטקסט "בדיקה ראשונה" (הקידומת `@מקובצקי` הוסרה אוטומטית).
- בעמודת `שם משימה מוצע` — "בדיקה ראשונה".

**אם נכשל:**
- אם הסקציה הצהובה לא צצה אבל ההודעה כן הגיעה ל-WhatsApp → ה-webhook לא רץ. בדוק Vercel logs לפונקציה `/api/webhooks/green-api`. ייתכן ש-`WHATSAPP_WEBHOOK_SECRET` לא תואם בין Vercel ל-Green API console.
- אם הסקציה צצה אבל בלי `suggestedTaskName` → `SYSTEM_WHATSAPP_NAME` או `SYSTEM_WHATSAPP_PHONE` לא נטענו. תוודא ב-`/settings/users` שאתה רואה שאתה אדמין (כי `audit-log` יראה את הסיבה), ולחילופין: דרשי ש-Vercel יעשה Redeploy של main כדי להחזיר את ה-env vars.
- אם בכלל לא הגיע ל-`/inbox` שום דבר → ייתכן ש-Green API לא ירה webhook. בדוק ב-Green API console את ה-history של ה-webhook.

**תוצאה צפויה:**

**תוצאה צפויה:**

**אם נכשל:**

- אם השליחה נכשלת עם 4xx/5xx — בדוק את ה-error באודיט (יישמר `event=whatsapp_group_send_failed`).

### 2.4 קליטת @mention עם קובץ (התסריט המרכזי)

**מטרה:** התסריט הכי חשוב — מישהו בקבוצה שולח קובץ לעבר המערכת.

**שלבים:**
1. ב-WhatsApp, צרף קובץ כלשהו לקבוצה (תמונה / PDF / מסמך — מה שיש לך).
2. בכיתוב (caption) של הקובץ, כתוב: `@מקובצקי טופס 4 חתום`
3. שלח. המתן ~10 שניות.

**תוצאה צפויה:**
- ב-`/projects/[id]/whatsapp` סקציית timeline — צצה שורה **נכנסת** חדשה:
  - חץ ↙ (נכנס)
  - שם השולח שלך + הטלפון שלך
  - 📎 שם הקובץ + סוג MIME
  - הכיתוב (אם נשאר אחרי הסרת התיוג — במקרה הזה כלום)
  - תווית כתומה: "🟡 הוצע: טופס 4 חתום" (suggestedTaskName פעל ✓)
  - תווית אפורה: "ממתין לטיפול"
- **קליק על השורה** → ניווט ל-`/inbox` (כי הסטטוס PENDING).
- ב-`/inbox` — הקובץ שם, עם:
  - עמודת `שם משימה מוצע` = "טופס 4 חתום"
  - עמודת `היתר משויך` = ההיתר הפעיל האחרון של הפרויקט (auto-assigned ✓ PR-3 §4.3)
  - לחיצה על שם הקובץ → תקבל URL חתום של 1h מ-Supabase Storage, הקובץ ניתן להורדה.

**אם נכשל:**
- אם הקובץ לא הגיע ל-/inbox → ה-webhook נכשל בהורדת המדיה. Vercel logs יראו `[green-api-webhook] processing failed` + סיבה.
- אם הוא הגיע אבל בלי suggestedTaskName → תיוג ה-`@מקובצקי` לא זוהה. בדוק את `SYSTEM_WHATSAPP_NAME` ב-Vercel ובדוק שב-WhatsApp הוקלד `@מקובצקי` ולא `@מקובצקי CRM` (במקרה השני ה-regex עדיין יעבוד אם השם המוקלד הוא רק "מקובצקי" — אם לא, אפשר לבדוק עם הטלפון: `@972539456995`).
- אם הוא הגיע בלי assignedPermitId → ייתכן שאין היתר פעיל בפרויקט. תיצור היתר ותחזור.
