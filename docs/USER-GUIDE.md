# מדריך מערכת — Makovetzky ERP

> מערכת ניהול היתרים, פרויקטים ותהליכים מול רשויות בענף הבנייה.
> כתובת ייצור: **https://makovetzky-erp.vercel.app**

---

## 1. סקירה כללית

המערכת בנויה סביב **היררכיה של 4 רמות** (אילוץ קשה — לא ניתן לעקוף):

```
לקוח (Client)
  └── פרויקט / עסקה (MasterDeal)
       └── היתר (Permit)              ← מרבית העבודה היומיומית מתבצעת ברמה זו
            └── בניין (Building)
```

משימות, milestones כספיים, מסמכים והערות תלויים **בהיתר** (Permit), לא ברמות הגבוהות יותר.

**מצבי היתר:** `DRAFT` → `IN_PROGRESS` → `AWAITING_AUTHORITY` → `COMPLETED` / `CANCELLED`
היתר ב-`COMPLETED` **נעול לעריכה** — לא ניתן להוסיף משימות, מסמכים, milestones או הערות. ניתן לפתוח מחדש (מנהל בלבד).

**מצבי משימה:** `OPEN` / `IN_PROGRESS` / `AWAITING_AUTHORITY` / `COMPLETED` / `BLOCKED`

**מצבי milestone כספי:** `PENDING` → `DUE` → `PAID`
המעבר ל-`DUE` מתרחש או כשהמשימה המעוגנת מסתיימת, או כשאחוז ההתקדמות של ההיתר חוצה את הסף שהוגדר (1–100%).

## 2. תפקידים והרשאות

### 🔑 ADMIN — מנהל
רואה הכל, משנה הכל. גישה ל-10 העמודים בתפריט.
- **לבד יכול:** מחיקות, ניהול משתמשים, ניהול לקוחות, ניהול ספקים, ניהול milestones כספיים, הגדרות, סל מיחזור, פעולות bulk על משימות, ייצוא/ייבוא CSV.
- **רואה כספים:** מחירי עסקאות, milestones, סטטוס תשלום, דוחות עמלות לספקים.
- **לא יכול:** להשבית את עצמו, להוריד את עצמו מ-ADMIN.

### 👤 EMPLOYEE — עובד
ניהול שוטף של היתרים ומשימות, **בלי גישה לכספים ולקוחות**.
- **יכול:** לעדכן משימות, להעלות מסמכים, לאשר מסמכים, ליצור הערות, ליצור Magic Links, לטפל בתיבת WhatsApp, ליצור היתרים ופרויקטים חדשים.
- **לא יכול:** לראות/לעדכן milestones כספיים, לראות לקוחות, לראות ספקים, לראות הגדרות.
- בתפריט: הפריטים האסורים מופיעים אפורים עם אייקון מנעול.

### 🤝 CONTRACTOR — קבלן/גורם חיצוני
משתמש בפורטל ייעודי (`/portal`), לא בלוח הבקרה הפנימי.
- **רואה:** רק לקוחות שיש לו `PortalAccess` אליהם, רק את ההיתרים והמשימות המוקצות לו.
- **יכול:** להעלות מסמכים למשימות שלו, לעדכן סטטוס משימות שלו.

### 🔓 PUBLIC — אורח עם קישור (ללא לוגין)
ארבעה זרימות ציבוריות, כל אחת עם הגנה ייחודית:
1. `/login` — התחברות בלבד.
2. `/m/[token]` — Magic Link לעובד שטח (48 שעות, חד-פעמי לכל משימה).
3. `/quote/[id]` — צפייה וחתימה דיגיטלית על הצעת מחיר.
4. `/api/webhooks/*` — קבלת הודעות מ-WhatsApp (דורש secret כ-header).

## 4. תהליכי עבודה עיקריים

### 4.1 פתיחת פרויקט חדש (Workflow מלא)
**מסך:** `/permits/new`

טופס אחד טרנזקציוני שיוצר במכה אחת: לקוח (אם חדש) → פרויקט (MasterDeal) → היתר → כל המשימות מהתבנית של הרשות+סוג-המבנה שנבחרו.

1. בחר/צור **לקוח** (חברה, איש קשר, טלפון)
2. תן שם **לפרויקט** ומחיר כולל
3. בחר **רשות** + **סוג מבנה** + **מספר היתר** (אופציונלי)
4. שלח — המערכת תיצור אוטומטית את כל המשימות מהתבנית, כולל תאריכי יעד מחושבים מ-`defaultDurationDays`, כולל תלויות בין משימות.

מי שמוקצה לכל משימה נקבע מ-`TaskTemplate.defaultAssigneeId` של התבנית (אם מוגדר). תאריך תחילה מחושב מ-`defaultStartOffsetDays`.

### 4.2 ניהול היתר — 5 לשוניות
**מסך:** `/permits/[id]/...`

- **משימות** (`/tasks`) — טבלה דחוסה + תצוגת Gantt (`?view=timeline`). מסננים לפי סטטוס, אחראי. צ'ק-בוקסים ל-bulk actions (שינוי אחראי / סטטוס / מחיקה).
- **כספים** (`/finances`) — milestones של ההיתר. הוספה/עריכה/סימון כשולם. עובד רואה במצב read-only.
- **מסמכים** (`/documents`) — העלאה (עד 25MB), אישור גרסה אחרונה, שיוך לבניין/משימה.
- **הערות** (`/notes`) — Markdown, ניתן לקבע (pin) הערות חשובות לראש הרשימה.
- **יומן ביקורת** (`/audit`) — Audit log של כל שינוי על ההיתר וילדיו (immutable).

### 4.3 Magic Links — שיתוף עם עובד שטח
**מסך:** טבלת משימות בכל היתר, אייקון 🔗 בשורת המשימה.

1. ADMIN/EMPLOYEE לוחץ על האייקון → דיאלוג נפתח עם URL חד-פעמי לתוקף של 48 שעות.
2. שולח את הקישור בוואטסאפ לעובד.
3. עובד פותח את הקישור בנייד — רואה את פרטי המשימה וטופס העלאת קובץ אחד.
4. הקובץ נוחת כ-Document במשימה עם `source: "MAGIC_LINK"`, ללא `uploadedById`.
5. עד שפג התוקף, הקישור ניתן לשימוש חוזר.

### 4.4 תיבת WhatsApp — שיוך מסמכים נכנסים
**מסך:** `/inbox`

- מסמכים שמתקבלים דרך webhook (Meta WhatsApp Cloud, Green API, או generic) נוחתים כאן כ-`PendingDocument` במצב `PENDING`.
- ADMIN/EMPLOYEE רואה את השולח, התוכן, ויכול:
  1. **לשייך** למשימה/היתר → הופך ל-`Document` רגיל, מעודכן ל-`ASSIGNED`.
  2. **לדחות** עם סיבה → סטטוס `REJECTED`.
- ניתן גם להעלות ידנית מסמך לתיבה (`sourceChannel: "MANUAL"`).

### 4.5 הצעות מחיר → המרה לפרויקט (Block 15)
**מסך:** `/proposals`

1. ADMIN יוצר טיוטה (`DRAFT`) ללא לקוח רשמי — רק שם לקוח, טלפון, מיקום, סכום, milestones לפי אחוזים.
2. שולח → `SENT` + קישור ציבורי `/quote/[id]`.
3. הלקוח חותם דיגיטלית → `SIGNED` + `signatureData` נשמר.
4. ADMIN לוחץ "המר לפרויקט" → המערכת יוצרת `Client` + `MasterDeal` + `DealMilestone`-ים מההצעה. סטטוס → `CONVERTED`.

### 4.6 מצב פרטיות בדשבורד (Block 19b)
דשבורד הבית מציג גם נתונים תפעוליים וגם כספיים. החלק הכספי **מוסתר כברירת מחדל** עם blur.
- כפתור "מוסתר — לחץ לחשיפה" / "חשוף — לחץ להסתרה" בראש הסקציה.
- ההגדרה נשמרת ב-localStorage, מסונכרנת בין טאבים.
- שווה במצב של ישיבה עם לקוח שעלול לראות את המסך.

### 4.7 פעולות Bulk על משימות (Block 20)
**מסך:** `/tasks` או טבלת משימות בכל היתר.

1. סמן צ'ק-בוקסים על משימות (יחיד או "סמן הכל").
2. סרגל צף מופיע למעלה: שינוי אחראי / שינוי סטטוס / מחיקה אדומה.
3. מחיקה חסומה אם אחת מהמשימות מעגנת milestone שטרם שולם.

### 4.8 סל מיחזור (Block 7b)
**מסך:** `/settings/recycle-bin`

- כל המחיקות במערכת הן **soft delete** (`deletedAt` מתעדכן, השורה לא נמחקת).
- המנהל יכול:
  - **לשחזר** פריט → `deletedAt = null`.
  - **למחוק לצמיתות** (purge) → מחיקה אמיתית עם cascades. חסום אם יש ילדים.

## 7. Audit Log — מי עשה מה ומתי

כל פעולה (CREATE / UPDATE / DELETE / STATUS_CHANGE / ASSIGN / APPROVE / REJECT) נכתבת ל-`AuditLog` בתוך אותה טרנזקציה.
- שדות נשמרים: `entityType`, `entityId`, `action`, `oldValue`, `newValue`, `userId`, `timestamp`.
- צפייה: לשונית "יומן ביקורת" בכל היתר.
- ה-log הוא **immutable by convention** — אין UI שמאפשר לערוך או למחוק רשומות ביקורת.

## 8. טיפים מהירים

- **קיצורי הסתרה:** מצב פרטיות הכספים מסונכרן בין כל הטאבים שלך — סגור מסך לפני שלקוח מציץ.
- **משימות עם תלויות שלא נסגרו:** סומנו "תלות לא ממומשת" — לא ניתן להזיז ל-`COMPLETED` בלי לפתור. מנהל יכול לעשות override.
- **תבניות:** הוסף `tags` (מופרדים בפס `|` ב-CSV) כדי לסנן משימות מהר.
- **מחיקה מקדימה:** משימה שמעגנת milestone לא משולם — לא נמחקת. שחרר את ה-milestone קודם.
- **לקוחות באנגלית:** לתוכן עברי מלא, השאר את שדה `companyName` כפי שהוא — המערכת לא מתרגמת.
- **כניסה מחדש להיתר שהושלם:** רק מנהל. נחשב כפעולה רגישה ונרשם ב-audit.
