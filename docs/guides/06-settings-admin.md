# הגדרות והרשאות

המדריך הזה מיועד ל-{role:admin}. כל העמודים כאן הם תחת `/settings/*` ודורשים תפקיד `ADMIN`. עובד רגיל לא רואה אותם בסיידבר.

## ניהול משתמשים — `/settings/users`

### יצירת משתמש

כפתור "+ משתמש חדש" פותח דיאלוג:

:::frame title="דיאלוג + משתמש חדש"
**+ משתמש חדש**

שם:        `_____________________` *
אימייל:    `_____________________` *
טלפון:     `_____________________`  ← *חיוני ל-WhatsApp ואיפוס סיסמה*

תפקיד:     [▾ EMPLOYEE]
&nbsp;&nbsp;&nbsp;⤷ {role:admin} / {role:employee} / {role:contractor}

סיסמה התחלתית: `_____________________` *
&nbsp;&nbsp;&nbsp;⤷ המשתמש יוכל לשנות מ-`/account/password`

[ ביטול ]   [ שמור ]
:::

### עריכת משתמש

כפתור **"ערוך"** בכל שורה. ניתן לעדכן את אותם השדות. **לא ניתן לעדכן סיסמה** — לזה יש כפתור נפרד.

### איפוס סיסמה (Admin Override)

כפתור **"אפס סיסמה"** ({role:admin} → משתמש אחר). פותח דיאלוג שבו מקלידים סיסמה חדשה והמשתמש יכול להתחבר איתה מיד. מתועד ב-AuditLog כ-`password_reset_by_admin`.

:::warning
זאת פעולה רגישה. השתמשו בה רק כשמשתמש איבד גישה ולא יכול לעשות `/forgot-password` (לדוגמה: אין לו טלפון מוגדר).
:::

### השבתה / הפעלה

כפתור **"השבת"** מסמן `isActive = false`. המשתמש לא יוכל להתחבר. הנתונים שלו (משימות שהוקצו, AuditLog) נשארים שלמים. הפעלה מחדש זמינה תמיד.

:::danger
**השבתה עצמית? אסורה.** אתם לא יכולים להשבית את עצמכם או להוריד את עצמכם מ-{role:admin}. מנגנון הגנה.
:::

## ניהול סיסמה אישית — `/account/password`

כל משתמש, ללא קשר לתפקיד, יכול להחליף סיסמה מעצמו:

1. הקלידו **סיסמה נוכחית** (אישור).
2. הקלידו **סיסמה חדשה** + אישור.
3. שמרו.

bcrypt משווה. שגיאה אם סיסמה נוכחית לא תואמת. מתועד ב-AuditLog.

## רשויות — `/settings/authorities`

הרשויות הן עיריות, ועדות מקומיות, גורמי בקרה (הג"א, כיבוי אש, יובלים). כל היתר במערכת מקושר לרשות בודדת.

### יצירה ועריכה

כפתור "+ רשות חדשה" → דיאלוג עם שלושה שדות: שם, מחוז, פרטי קשר. ערכו/מחקו מהשורה.

:::warning
**אי אפשר למחוק** רשות אם יש לה היתרים פעילים או תבניות משימות. הכפתור יהיה אפור עם tooltip.
:::

### Authority Wiki

ברשות יש **count Wiki**. לחיצה עליו פותחת `/settings/authorities/[id]` — דף עם רשומות Wiki בפורמט Markdown. כאן רושמים את הדרישות, הסטנדרטים, קישורים לטפסים, התנאים של אותה רשות.

:::frame title="Authority Wiki — דוגמת רשומה"
**ועדה מקומית ראשון לציון** — מחוז: דרום

📖 **רשומות Wiki** (5)

📄 **מסמכים נדרשים לטופס 4 — צוות מים**
   *עודכן 18/05/2026 ע"י איציק*

📄 **דרישות הג"א — בניין מגורים מעל 5 קומות**
   *עודכן 02/05/2026 ע"י איציק*

📄 **טפסים נדרשים לחתימת מהנדס**
   *עודכן 15/04/2026 ע"י איציק*

[ + רשומה חדשה ]
:::

:::tip
**רשומה טיפוסית:** "מסמכים נדרשים לטופס 4 — צוות מים", "דרישות הג"א לבניין מגורים מעל 5 קומות", "טפסים נדרשים לחתימת מהנדס".
:::

## סוגי בניינים — `/settings/building-types`

מערכת התבניות עובדת על צירוף **רשות × סוג בניין**. כל היתר חדש מקבל את המשימות שמתאימות לצירוף שלו.

:::frame title="דוגמת מטריצה — איך עובדות התבניות"
| רשות \ סוג בניין | וילה פרטית | מגדל מגורים |
|---|---|---|
| ועדה מקומית ראשל"צ | 12 תבניות | 28 תבניות |
| ועדה מקומית תל-אביב | 18 תבניות | 35 תבניות |
| ועדה מקומית הרצליה | 14 תבניות | 24 תבניות |
:::

כאן מנהלים את רשימת **סוגי הבניינים**. שדות: שם, תיאור. אי אפשר למחוק אם יש תבניות שמשתמשות בו.

## תבניות משימות — `/settings/templates`

זה **הקסם של המערכת**. תבנית = "משימה שתופיע אוטומטית בכל היתר חדש שמתאים".

### בחירת הצירוף

ראשית בוחרים רשות + סוג בניין. הטבלה מציגה את כל התבניות לצירוף.

### יצירה / עריכת תבנית

כפתור "+ תבנית חדשה". שדות:

| שדה | תפקיד |
|---|---|
| שם תבנית | מעבר ישיר ל-`Task.name` בעת היצירה |
| תיאור | ל-`Task.description` |
| משך ימים ברירת מחדל | תאריך יעד = יצירה + N |
| סדר עדיפות | קטן יותר = למעלה בטבלה |
| **קטגוריה** | קובע את ה-pill ההגשה (קריטי!) |
| אחריות | פנימי / לקוח / קבלן / רשות |
| תגיות | לחיפוש |
| **אחראי ברירת מחדל** | המשתמש שיוקצה אוטומטית |
| פעיל | אם false — לא ייווצרו ממנו משימות חדשות |

### ניהול תלויות

עמודת "תלויות" בטבלה. לחיצה פותחת **Dependency Manager** — בחירת תבניות אחרות שהמשימה תלויה בהן.

:::frame title="Dependency Manager — תבנית [טפסים] חתימת מהנדס"
**תלויה ב:**
☑ [הג"א] תוכנית מקלט
☑ [יובלים] חיבור מים
☐ [כיבוי אש] אישור מתזים
☐ [טפסים] מדידת מצב קיים

→ המשימה תהיה **חסומה** עד שכל הסימוניות יושלמו
:::

:::info
**גרפי תלויות מעוגלים נחסמים אוטומטית** — המערכת לא תאפשר תלות שתיצור מעגל.
:::

### ייבוא / ייצוא CSV

תבניות הן הרבה. כפתור "ייצא CSV" מוריד את כל התבניות של הצירוף הנוכחי. "ייבא CSV" מקבל קובץ ויוצר/מעדכן. שימושי להעברת תבניות בין צירופים.

## סל מיחזור — `/settings/recycle-bin`

כל ישויות המערכת תומכות ב-**soft-delete** דרך עמודת `deletedAt`. מחיקה לא מוחקת מה-DB — היא רק מסמנת את הרשומה כמחוקה ומסתירה אותה.

:::frame title="/settings/recycle-bin"
**🗑 סל מיחזור**

[ היתרים (3) ] [ משימות (12) ] [ מסמכים (5) ] [ הערות (2) ] [ אבני דרך (1) ]

──────────────────────────────────────────
📜 **2025/394 — מגדל בודד — היתר ראשי**
מחוק ע"י איציק · 02/05/2026

[ ⤴ שחזר ]   [ ⛔ מחק לצמיתות ]
──────────────────────────────────────────
📜 **2025/401 — בודק — בדיקה**
מחוק ע"י איציק · 14/04/2026

[ ⤴ שחזר ]   [ ⛔ מחק לצמיתות ]
:::

לכל רשומה שתי פעולות:

- **⤴ שחזר** — מאפס `deletedAt` ל-null. הישות חוזרת.
- **⛔ מחק לצמיתות** — `DELETE` קשיח מה-DB. **אי הפיכה**. אם זה מסמך, גם הקובץ נמחק מ-Supabase Storage.

### Cascade — מחיקה מוטלת בריצה

כשמוחקים היתר, ה-cascade מסמן גם את כל המשימות + המסמכים שלו כ-`deletedAt`. כשמוחקים פרויקט (MasterDeal), ה-cascade עובר על כל ההיתרים שלו ועליהם על כל הילדים.

:::warning
**שחזור הוא פר רשומה.** אם תרצו להחזיר היתר, תצטרכו לשחזר ידנית גם את המשימות שלו.
:::

## יומן פעולות — `/settings/audit-log`

עמוד גלריה לכל פעולה במערכת. כל פעולה שמשנה DB (יצירה / עדכון / מחיקה / שינוי סטטוס / השלמה / דחייה) כותבת רשומה ל-`AuditLog`.

### 5 מסננים בעמוד

| מסנן | אפשרויות |
|---|---|
| סוג ישות | Permit / Task / Document / ... |
| מזהה ישות | כל הפעולות על משימה X |
| פעולה | CREATE / UPDATE / DELETE / STATUS_CHANGE / ... |
| משתמש | מי עשה |
| טווח תאריכים | מ-עד |

כל המסננים URL-encoded. 50 רשומות לעמוד עם pagination.

### תצוגת JSON Diff

כל שורה מציגה "שדות: a · b · c" — סיכום של השדות שהשתנו. לחיצה פותחת תצוגה מורחבת עם old/new מקובלים LTR pretty-printed.

:::frame title="Audit Log — דוגמת רשומה מורחבת"
🟦 **STATUS_CHANGE** · Task #847
*ע"י איציק · 04/06/2026 16:42*

שדות שהשתנו: `status`, `completedAt`

```
- status:       "IN_PROGRESS"
+ status:       "COMPLETED"

- completedAt:  null
+ completedAt:  "2026-06-04T16:42:13Z"
```
:::

:::tip
**דוגמת שימוש:** משימה הופנתה לחזרה ל-OPEN מ-COMPLETED מבלי שאתם זוכרים. סננו AuditLog לפי ה-entityId של המשימה → תראו את ה-STATUS_CHANGE עם המשתמש והערכים.
:::

:::danger
**Audit Log לעולם לא נמחק.** אין endpoint למחיקת או עדכון רשומות AuditLog. הוא immutable מבחינת הקוד.
:::

## הגדרות WhatsApp — `/settings/whatsapp`

כרגע עמוד placeholder. בעתיד יכלול חיבור Green API, סטטוס webhook, חיבור קבוצות.

## טיפים ל-{role:admin}

:::tip
**בנו תבניות לפני שאתם מוסיפים פרויקטים.** היתרים בלי תבניות נוצרים ריקים, ותצטרכו ידנית להוסיף את כל המשימות.
:::

:::tip
**שמרו `defaultAssignee` בתבניות.** כל משימה תהיה למישהו ב-`/my-tasks` שלו. בלעדיו, המשימה תהיה "יתומה".
:::

:::tip
**ה-Wiki לרשות הוא הזיכרון הארגוני שלכם.** אל תחסכו בו. רשמו הכל שגיליתם.
:::

:::tip
**AuditLog הוא ידידכם בכל ספק.** לפני שאתם מאשימים מישהו במשהו — סננו ובדקו.
:::

:::info
**סל המיחזור בטוח.** מחיקה רכה היא הפיכה. עזרו לעצמכם, אל תפחדו למחוק.
:::

:::warning
**בעת מחיקת לקוח** — cascade יוצר היררכיה: מוחקים גם MasterDeals → היתרים → משימות. הדיאלוג יציג את הספירה לפני שתאשרו.
:::
