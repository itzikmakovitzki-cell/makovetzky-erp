# הגדרות והרשאות

המדריך הזה מיועד למנהל. כל העמודים כאן הם תחת `/settings/*` ודורשים תפקיד `ADMIN`. עובד רגיל לא רואה אותם בסיידבר.

## ניהול משתמשים — `/settings/users`

### יצירת משתמש

כפתור "+ משתמש חדש" פותח דיאלוג:

| שדה | חובה | הערה |
|---|---|---|
| שם | ✓ | מופיע בכל מקום |
| אימייל | ✓ | משמש להתחברות |
| **טלפון** | מומלץ | חיוני לתזכורות WhatsApp ולאיפוס סיסמה |
| תפקיד | ✓ | `ADMIN` / `EMPLOYEE` / `CONTRACTOR` |
| סיסמה התחלתית | ✓ | המשתמש יוכל לשנות מ-`/account/password` |

### עריכת משתמש

כפתור **"ערוך"** בכל שורה. ניתן לעדכן את אותם השדות. לא ניתן לעדכן סיסמה — לזה יש כפתור נפרד.

### איפוס סיסמה (Admin Override)

כפתור **"אפס סיסמה"** (אדמין → משתמש אחר). פותח דיאלוג שבו אדמין מקליד סיסמה חדשה והמשתמש יכול להתחבר איתה מיד. מתועד ב-AuditLog כ-`password_reset_by_admin`.

> ⚠️ זאת פעולה רגישה. השתמש בה רק כשמשתמש איבד גישה ולא יכול לעשות `/forgot-password` (לדוגמה: אין לו טלפון מוגדר).

### השבתה / הפעלה

כפתור **"השבת"** מסמן `isActive = false`. המשתמש לא יוכל להתחבר. הנתונים שלו (משימות שהוקצו לו, AuditLog) נשארים שלמים. הפעלה מחדש זמינה תמיד.

### השבתה עצמית? אסורה

אתה לא יכול להשבית את עצמך או להוריד את עצמך מ-ADMIN. מנגנון הגנה.

## ניהול סיסמה אישית — `/account/password`

כל משתמש, ללא קשר לתפקיד, יכול להחליף סיסמה מעצמו ב-`/account/password`:

1. הקלד **סיסמה נוכחית** (אישור).
2. הקלד **סיסמה חדשה** + אישור.
3. שמור.

ה-bcrypt משווה. שגיאה אם סיסמה נוכחית לא תואמת. מתועד ב-AuditLog.

## רשויות — `/settings/authorities`

הרשויות הן עיריות, ועדות מקומיות, גורמי בקרה (הג"א, כיבוי אש, יובלים). כל היתר במערכת מקושר לרשות בודדת.

### יצירה ועריכה

כפתור "+ רשות חדשה" פותח דיאלוג עם שלושה שדות: שם, מחוז, פרטי קשר. ערוך/מחק מהשורה.

**אי אפשר למחוק** רשות אם יש לה היתרים פעילים או תבניות משימות. הכפתור יהיה אפור עם tooltip.

### Authority Wiki

ברשות יש **count Wiki** ולחיצה עליו פותחת `/settings/authorities/[id]` — דף עם רשומות Wiki בפורמט Markdown. כאן רושמים את הדרישות, הסטנדרטים, קישורים לטפסים, התנאים של אותה רשות. ניתן ליצור / לערוך / למחוק רשומות.

ה-Markdown מוצג עם renderer מובנה (no deps) שתומך ב-headings, רשימות, קישורים, mono.

> 💡 רשומת Wiki טיפוסית: "מסמכים נדרשים לטופס 4 — מצוות מים", "דרישות הג"א לבניין מגורים מעל 5 קומות", "טפסים נדרשים לחתימה ע"י מהנדס".

## סוגי בניינים — `/settings/building-types`

מערכת התבניות עובדת על צירוף **רשות × סוג בניין**. כל היתר חדש מקבל את המשימות שמתאימות לצירוף שלו.

לדוגמה:
- ועדה מקומית ראשון לציון × וילה פרטית → תבניות ספציפיות
- ועדה מקומית ראשון לציון × מגדל מגורים → תבניות אחרות
- ועדה מקומית תל-אביב × וילה פרטית → תבניות אחרות

כאן מנהלים את רשימת **סוגי הבניינים**. שדות: שם, תיאור. אי אפשר למחוק אם יש תבניות שמשתמשות בו.

## תבניות משימות — `/settings/templates`

זה הקסם של המערכת. תבנית = "משימה שתופיע אוטומטית בכל היתר חדש שמתאים".

### בחירת הצירוף

ראשית בוחרים רשות + סוג בניין. הטבלה מציגה את כל התבניות לצירוף הזה.

### יצירה / עריכה תבנית

כפתור "+ תבנית חדשה". שדות:

| שדה | תפקיד |
|---|---|
| שם תבנית | מעבר ישיר ל-`Task.name` בעת היצירה |
| תיאור | ל-`Task.description` |
| משך ימים ברירת מחדל | תאריך יעד = יצירה + N ימים |
| סדר עדיפות | סדר בטבלת המשימות (קטן יותר = למעלה) |
| **קטגוריה** | חיוני! קובע את ה-pill ההגשה |
| אחריות | פנימי / לקוח / קבלן / רשות |
| תגיות (`tags`) | לחיפוש |
| **אחראי ברירת מחדל** | המשתמש שיוקצה אוטומטית |
| פעיל | אם false — לא ייווצרו ממנו משימות חדשות (אבל לא ימחק רטרואקטיבית) |

### ניהול תלויות

עמודת "תלויות" בטבלה. לחיצה פותחת **Dependency Manager** — בחירת תבניות אחרות שהמשימה תלויה בהן. בעת יצירת היתר, המערכת בונה את הגרף של `TaskDependency` בהתאמה (אחרי שיוצרת את כל המשימות מהתבניות).

> 🔁 גרפי תלויות מעוגלים נחסמים אוטומטית — המערכת לא תאפשר תלות שתיצור מעגל.

### ייבוא / ייצוא CSV

תבניות הן הרבה. כפתור "ייצא CSV" מוריד את כל התבניות של הצירוף הנוכחי. "ייבא CSV" מקבל קובץ ויוצר/מעדכן. שימושי להעברת תבניות בין צירופים.

## סל מיחזור — `/settings/recycle-bin`

מאז Block 7b, כל ישויות המערכת תומכות ב-**soft-delete** דרך עמודת `deletedAt`. מחיקה לא מוחקת מה-DB — היא רק מסמנת את הרשומה כמחוקה ומסתירה אותה מהתצוגות הרגילות.

ב-`/settings/recycle-bin` רואים את כל מה שמחוק לפי סוג:

- היתרים מחוקים
- משימות מחוקות
- מסמכים מחוקים
- הערות מחוקות
- אבני דרך מחוקות

לכל רשומה שתי פעולות:

- **שחזר** — מאפס `deletedAt` ל-null. הישות חוזרת.
- **מחק לצמיתות** — `DELETE` קשיח מה-DB. אי הפיכה. אם זה מסמך, גם הקובץ נמחק מ-Supabase Storage.

### Cascade — מחיקה מוטלת בריצה

כשמוחקים היתר, ה-cascade מסמן גם את כל המשימות + המסמכים שלו כ-`deletedAt`. כשמוחקים פרויקט (MasterDeal), ה-cascade עובר על כל ההיתרים שלו ועליהם על כל הילדים. שחזור הוא **פר רשומה** — אם תרצו להחזיר היתר, תצטרכו לשחזר ידנית גם את המשימות שלו.

## יומן פעולות — `/settings/audit-log` (PR #45)

עמוד גלריה לכל פעולה במערכת. כל פעולה שמשנה DB (יצירה / עדכון / מחיקה / שינוי סטטוס / השלמה / דחייה) כותבת רשומה ל-`AuditLog` עם:

- timestamp
- entityType + entityId
- action (`CREATE` / `UPDATE` / `DELETE` / `STATUS_CHANGE` / `ASSIGN` / `DEPENDENCY_OVERRIDE` / `APPROVE` / `REJECT`)
- oldValue (JSON)
- newValue (JSON)
- userId (מי עשה)

### 5 מסננים בעמוד

- סוג ישות (Permit / Task / Document / ...)
- מזהה ישות (לדוגמה: כל הפעולות על משימה X)
- פעולה
- משתמש
- טווח תאריכים (מ-עד)

כל המסננים URL-encoded. 50 רשומות לעמוד עם pagination.

### תצוגת JSON Diff

כל שורה מציגה "שדות: a · b · c" — סיכום של השדות שהשתנו. לחיצה פותחת תצוגה מורחבת עם old/new מקובלים LTR pretty-printed.

> 🔍 **שימוש לדוגמה:** משימה הופנתה לחזרה ל-OPEN מ-COMPLETED מבלי שאתה זוכר. סנן AuditLog לפי ה-entityId של המשימה → תראה את ה-STATUS_CHANGE עם המשתמש והערכים.

### Audit Log לעולם לא נמחק

לפי קונבנציה, אין endpoint למחיקת או עדכון רשומות AuditLog. הוא **immutable** (לא enforced ב-DB, אבל אין שום קוד שכותב UPDATE/DELETE עליו).

## הגדרות WhatsApp — `/settings/whatsapp`

כרגע עמוד placeholder. בעתיד יכלול חיבור Green API, סטטוס webhook, חיבור קבוצות.

## טיפים לאדמין

- **בנו תבניות לפני שאתם מוסיפים פרויקטים** — היתרים בלי תבניות נוצרים ריקים, ותצטרכו ידנית להוסיף את כל המשימות.
- **שמרו `defaultAssignee` בתבניות** — כל משימה תהיה ל-מישהו ב-`/my-tasks` שלו. בלעדיו, המשימה תהיה "יתומה".
- **ה-Wiki לרשות הוא הזיכרון הארגוני שלכם** — אל תחסכו בו. רשמו הכל שגיליתם.
- **AuditLog הוא ידידכם בכל ספק** — לפני שאתם מאשימים מישהו במשהו, סננו ובדקו.
- **סל המיחזור הוא בטוח** — מחיקה רכה היא reversible. עזרו לעצמכם, אל תפחדו למחוק.
- **בעת מחיקת לקוח** — cascade יוצר היררכיה: מוחקים גם MasterDeals → היתרים → משימות. הdialog יציג את הספירה לפני שתאשרו.
