# טופס 4 — תהליך מקצה לקצה

זה התהליך הראשי במערכת — איך מוציאים היתר טופס 4 / תעודת גמר מהרגע שנקבע פרויקט עד הסגירה הסופית.
כל שלב כאן ממופה לפעולה במערכת.

:::info
**מי המדריך הזה?** {role:admin} ו-{role:employee} משתמשים בו יחד. השלבים שדורשים אדמין בלבד (אבני דרך, מחיקות) מסומנים בצ׳יפ {role:admin} ליד הכותרת.
:::

## שלב 0 — הכנה (פעם אחת) {role:admin}

לפני שיוצרים פרויקט, ההגדרות הבאות צריכות להיות מסודרות:

1. **רשות** — ב-`/settings/authorities` מוסיפים את הרשות הרלוונטית (לדוגמה: "ועדה מקומית ראשון לציון") + רשומות Wiki עם דרישות הסטנדרט שלה.
2. **סוג בניין** — ב-`/settings/building-types` מוסיפים את סוג הבניין (לדוגמה: "מגדל מגורים").
3. **תבניות משימות** — ב-`/settings/templates` בוחרים רשות + סוג בניין → מגדירים את כל המשימות שצריך לעשות עבור הצירוף הזה.

כל תבנית כוללת:

| שדה | תפקיד |
|---|---|
| שם משימה | לדוגמה: `[הג"א] תוכנית מקלט` |
| תיאור | הסבר חופשי |
| משך ימים | תאריך יעד מחושב = יצירה + N |
| סדר עדיפות | סדר בטבלה |
| קטגוריה | קובע את ה-pill ההגשה |
| אחריות | פנימי / לקוח / קבלן / רשות |
| אחראי ברירת מחדל | יוקצה אוטומטית |
| תלויות | במשימות אחרות |

:::success
**הקסם של המערכת:** מרגע שהתבניות מוגדרות, **כל היתר חדש לאותה רשות + סוג בניין יקבל את כל המשימות אוטומטית** עם תאריכי יעד מחושבים ותלויות מוגדרות. אתם משקיעים פעם אחת ונהנים לנצח.
:::

## שלב 1 — יצירת הפרויקט וההיתר

ב-`/permits/new` יש טופס יחיד שיוצר במכה: לקוח (אם חדש) + פרויקט + היתר + בניינים.

:::frame title="/permits/new — הטופס המאוחד"

**📁 פרטי הפרויקט**

&nbsp;&nbsp;&nbsp;תאריך התחלה צפוי: `[2026-06-04]`

**📜 פרטי ההיתר**
&nbsp;&nbsp;&nbsp;שם היתר: `_____________________`
&nbsp;&nbsp;&nbsp;מספר היתר: `_____________________`
&nbsp;&nbsp;&nbsp;רשות: [▾ ועדה מקומית ראשון לציון]

**🏢 בניינים**

לאחר השמירה, ההיתר נפתח עם 5 לשוניות. הלשונית הראשונה (משימות) תהיה כבר מלאה במשימות מהתבניות.

### יצירת היתר נוסף לפרויקט קיים

ב-`/projects/[id]`, כפתור **"הוסף היתר"** פותח דיאלוג שיוצר היתר חדש מתחת לעסקה הקיימת — בלי להזין מחדש את הלקוח. שימושי לפרויקטים עם כמה היתרים נפרדים.

## שלב 2 — ניהול משימות

לשונית **משימות** של ההיתר היא היומיום של ה-PM. שתי תצוגות:

- **טבלה** (ברירת מחדל בדסקטופ) — צפופה, עם פס סטטוס צבעוני שמאלה לכל שורה, עריכה בשורה.
- **Gantt / Timeline** — `?view=timeline` בכתובת. תצוגת ציר זמן עם מקרא צבעים.
- **כרטיסים במובייל** — אותו תוכן ופקדים, רק חברתי יותר לאצבע.

### צבעי פס הסטטוס — קריאה במבט אחד

:::frame title="טבלת משימות — דוגמה לפסים צבעוניים"
🔴 **[הג"א] תוכנית מקלט** — איחור 3 ימים · אחראי: יוסי · יעד: 01/06 ⏰ נדחה 2x
🟡 **[כיבוי אש] אישור מערכת מתזים** — ממתין לרשות · אחראי: שרה · יעד: 15/06
⚫ **[טפסים] חתימת מהנדס** — חסום (תלוי בהגשת הג"א) · אחראי: דני
🟢 **[יובלים] חיבור מים** — הושלם · אחראי: משה · יעד: 28/05 ✓
:::

| צבע | משמעות |
|---|---|
| 🔴 אדום | באיחור — תאריך יעד עבר, לא הושלם, לא מוקפא |
| 🟡 כתום | ממתין לרשות (`AWAITING_AUTHORITY`) או מוקפא |
| ⚫ אפור | חסום ע"י תלות |
| 🟢 ירוק | הושלם |

### עדיפויות וזרקור

- **🌟 זרקור (Spotlight)** — סימון לחיצה אחת על כוכב. משימות עם זרקור עולות לראש הרשימה ובדשבורד.
- **דחוף (URGENT)** — תג אדום בולט. דורש טיפול מיידי.

:::warning
**זרקור — לשימוש זהיר.** אם הכל בזרקור, שום דבר לא בזרקור. שמרו את זה ל-3-5 משימות שבאמת לוהטות.
:::

### עריכה מהירה (Inline)

על כל משימה אפשר לערוך **בשורה** — בלי לפתוח דף:

- **סטטוס** — לחיצה על ה-pill פותחת dropdown.
- **אחראי** — לחיצה על שם האחראי פותחת בחירת משתמש.
- **תאריך יעד** — לחיצה פותחת date picker.

כל שינוי נשמר אוטומטית עם spinner קטן ונכתב ל-Audit Log.

### Snooze (דחייה)

כפתור ⏰ ליד המשימה: **"דחה למחר"** / **"דחה לשבוע הבא"**.

- מזיז את תאריך היעד קדימה (מהתאריך המאוחר יותר — היום או היעד הנוכחי).
- מציג badge אדום **"נדחה N פעמים"** ליד היעד.
- ה-badge נשאר ומציק במכוון — דחיות חוזרות = משימה שצריך לדבר עליה.

### פעולות bulk {role:admin}

צ׳ק-בוקסים מימין לכל משימה + צ׳ק-בוקס "סמן הכל" בראש. כשנבחרת לפחות משימה אחת, צץ סרגל צף עם:

- שנה אחראי לכמה משימות במכה
- שנה סטטוס לכמה במכה
- מחק (חסום אם משימה מעוגנת ל-Milestone לא משולם)

## שלב 3 — מסמכים ו-Magic Links

לשונית **מסמכים** של ההיתר מציגה את כל המסמכים שהועלו, מסוננים לפי בניין / משימה / גרסה.

### העלאה ידנית

לחצן "העלה מסמך" פותח דיאלוג עם בורר קובץ + שדות מטא (משימה, בניין, הערה).

### Magic Link — עובד שטח בלי לוגין

על כל משימה — כפתור 🔗. פותח דיאלוג עם קישור ייחודי (token 32-תווי, תוקף 48 שעות).

:::frame title="דיאלוג Magic Link — מה שולחים לעובד"
**🔗 קישור העלאה למשימה**

`https://crm.makomedia.co.il/m/a7f3b9...c2d`

📋 [העתק קישור] 📱 [שלח ב-WhatsApp]

⏱ תוקף: 48 שעות · 🔄 מרובה-שימוש עד פקיעה
:::

הקישור פותח דף ציבורי שמראה את פרטי המשימה + טופס העלאת קובץ אנונימי. הקבצים נוחתים כ-`Document` עם `source: "MAGIC_LINK"`.

### אישור מסמך {role:admin}

מסמך שעבר אישור (`isLatestApproved = true`) מסומן בירוק. רק אדמין יכול לאשר.

## שלב 4 — הגשה לרשות

בראש לשונית המשימות יש **רצועת הגשות** — pill לכל קטגוריה:

:::frame title="רצועת הגשות בראש לשונית משימות"
[🟢 הג"א · APPROVED · 5/5]   [🟡 כיבוי אש · SUBMITTED · 4/4]   [⚪ יובלים · PREPARING · 3/7]   [🔴 טפסים · REJECTED · 2/4]
:::

לחיצה על pill פותחת דיאלוג מעבר סטטוס.

### לוגיקה של מעברי סטטוס

| מעבר | מה קורה |
|---|---|
| PREPARING → SUBMITTED | מקפיא את כל המשימות הפתוחות לקטגוריה ל-`AWAITING_AUTHORITY` 🟡 |
| SUBMITTED → APPROVED | **לא** מבטל הקפאה אוטומטית. אדמין מסמן ידנית COMPLETED |
| SUBMITTED → REJECTED | מבטל הקפאה. משימות חוזרות ל-OPEN לתיקון והגשה מחדש |
| רולבק SUBMITTED → PREPARING | מבטל הקפאה. סימטריה מלאה |

:::tip
**"מוכן להגשה" בדשבורד {role:admin}** — בעמוד `/` יש פאנל שמציג כל (היתר × קטגוריה) שבהן כל המשימות הושלמו אבל הקטגוריה עדיין לא הוגשה. לחיצה מובילה ישר לטבלת המשימות מסוננת, מוכן ללחוץ SUBMITTED.
:::

## שלב 5 — אבני דרך לחיוב {role:admin}

לשונית **כספים** של ההיתר נפתחת דרך כפתור "ניהול פיננסי". כאן יוצרים אבני דרך לחיוב.

עיין במדריך [כספים](/guides/finances) לפרטים מלאים.

## שלב 6 — סגירת ההיתר {role:admin}

כשכל המשימות הושלמו וכל הקטגוריות `APPROVED`:

1. סימון ההיתר כ-`COMPLETED`.
2. ההיתר **נעול לעריכה** — אי אפשר להוסיף משימות / מסמכים / אבני דרך / הערות.
3. ניתן לפתוח מחדש בכל עת מתפריט שורה.
4. `/permits?archived=1` כולל tab "הושלמו" — מסתיר אותם מהתצוגה הראשית.

:::danger
ההיתר נעול לעריכה לאחר השלמה. אם משכחתם להעלות מסמך — תצטרכו לפתוח מחדש קודם, ואז לסגור שוב. זה במכוון — מונע "נדמיין שזה הסתיים".
:::

## שלב 7 — סיכום והדפסה

בדף הפרויקט, לינק **"סיכום להדפסה"** מוביל ל-`/projects/[id]/print` — דף בעמוד אחד עם:

- זיהוי מלא של הלקוח
- סטטוס הפרויקט
- התקדמות פר היתר, משימות מקובצות לקטגוריה
- אבני דרך לחיוב {role:admin}
- ספר עמלות מספקים {role:admin}

הדפסה דרך <kbd>Ctrl+P</kbd> → "שמור כ-PDF". הכל מוכן ללקוח.

## טיפים שיחסכו לכם זמן

:::success
**1. קבעו תבניות פעם אחת.** כל היתר חדש מקבל את כל המשימות אוטומטית. רוב הזמן של "הוצאת היתר" הוא בפעם הראשונה.
:::

:::success
**2. השתמשו בקטגוריות.** לא רק לסיווג, אלא כיחידת ההגשה לרשות. כל קטגוריה = pill ברצועה.
:::

:::success
**3. הקפיאו ידנית** משימות שהוגשו לרשות גם אם לא הקפצתם את כל הקטגוריה.
:::

:::success
**4. Magic Links במקום משתמשים.** שלחו לעובדים בשטח — מהיר יותר, פחות תחזוקה.
:::

:::success
**5. Excel אחרי שינויים גדולים.** ייצאו ושלחו ללקוח לידוע אוטומטי.
:::
