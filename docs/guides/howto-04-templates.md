# איך מגדירים תבניות משימות

מדריך כיס — איך בונים את "המנוע" שייצור משימות אוטומטית בכל היתר חדש.

:::info
**מי יכול?** {role:admin} בלבד. ב-`/settings/templates`.
**למה זה חשוב?** היתר ללא תבניות נוצר ריק. עם תבניות — הוא נוצר מלא ועם תאריכי יעד מחושבים.
:::

## עקרון הצירוף

המערכת מחזיקה תבניות לפי **צירוף שני שדות**:

:::frame title="הצירוף = רשות × סוג בניין"
**ועדה מקומית ראשון לציון** × **מגדל מגורים** = רשימת תבניות א
**ועדה מקומית ראשון לציון** × **וילה פרטית** = רשימת תבניות ב
**ועדה מקומית תל-אביב** × **מגדל מגורים** = רשימת תבניות ג
:::

כל היתר חדש בודק את הרשות + סוג הבניין שלו, ושולף את התבניות המתאימות.

## השלבים — תבנית חדשה

1. כנסו ל-`/settings/templates`.
2. **בחרו רשות** מה-dropdown העליון.
3. **בחרו סוג בניין** מה-dropdown השני.
4. הטבלה תציג את התבניות הקיימות לצירוף. לחצו **"+ תבנית חדשה"**.
5. מלאו את השדות:

| שדה | המלצה |
|---|---|
| שם תבנית | התחילו ב-`[קטגוריה]`, לדוגמה `[הג"א] תוכנית מקלט` |
| תיאור | מה בדיוק צריך לעשות במשימה |
| משך ימים | כמה ימים מהיום שייווצרה המשימה ועד היעד |
| סדר עדיפות | קטן יותר = למעלה בטבלה |
| **קטגוריה** | חיוני! קובע את ה-pill ההגשה |
| אחריות | פנימי / לקוח / קבלן / רשות |
| **אחראי ברירת מחדל** | מי יקבל את המשימה כברירת מחדל |
| תגיות | לחיפוש (חופשי) |
| פעיל | ☑ — אחרת לא ייווצרו ממנו משימות חדשות |

6. שמרו.

## הוספת תלויות

לאחר יצירת התבנית, לחצו על העמודה "תלויות" בשורת התבנית כדי לפתוח **Dependency Manager**:

:::frame title="Dependency Manager — תבנית [טפסים] חתימת מהנדס"
**תלויה ב:**
☑ [הג"א] תוכנית מקלט
☑ [יובלים] חיבור מים
☐ [כיבוי אש] אישור מתזים
☐ [טפסים] מדידת מצב קיים
:::

המערכת תיצור `TaskDependency` בכל היתר חדש בהתאם — המשימה תהיה **חסומה ⚫** עד שהתלויות יושלמו.

:::info
**גרפי תלויות מעוגלים נחסמים אוטומטית** — לא תוכלו ליצור תלות שתיצור מעגל.
:::

## ייבוא / ייצוא CSV

תבניות הן הרבה. במקום להזין אותן אחת-אחת:

- **"ייצא CSV"** — מוריד את כל התבניות של הצירוף הנוכחי.
- **"ייבא CSV"** — מקבל קובץ ויוצר/מעדכן בהתאם.

זה שימושי גם להעברת תבניות בין צירופים (למשל מ"ראשל"צ × וילה" ל"תל-אביב × וילה" עם עדכונים נקודתיים).

## טיפים

:::tip
**ה-`defaultAssignee` קריטי.** משימות בלי אחראי הן "יתומות" — לא יופיעו ב-`/my-tasks` של אף אחד. הגדירו אחראי ברירת מחדל בכל תבנית.
:::

:::tip
**עיצבו את הקטגוריה לפי שלב ההגשה לרשות**, לא לפי סוג העבודה. לדוגמה — `הג"א`, `כיבוי אש`, `יובלים` הם קטגוריות טובות. `הכנת מסמכים` או `בדיקות` — פחות.
:::

:::warning
**שינוי תבנית לא משפיע רטרואקטיבית.** אם תוסיפו תבנית חדשה לצירוף — היתרים קיימים לא יקבלו את המשימה. רק היתרים חדשים שייווצרו אחרי השינוי.
:::

מדריך מלא בנושא: [הגדרות והרשאות](/guides/settings-admin).
