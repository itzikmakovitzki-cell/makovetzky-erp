# איך יוצרים אבן דרך לחיוב

מדריך כיס — איך מגדירים מתי לחייב את הלקוח.

:::info
**מי יכול?** {role:admin} בלבד.
**איפה?** בלשונית "כספים" של ההיתר (פותחים מ-Sheet "ניהול פיננסי").
:::

## שני סוגים — איזה לבחור?

:::cols
**🎯 מבוסס משימה**
ה-pill עוברת ל-DUE כשמשימה ספציפית מסומנת COMPLETED.

**טוב ל:**
- מקדמה (בחתימת חוזה)
- תשלום ביניים (בקבלת אישור הג"א)
- תשלום סופי (בקבלת טופס 4)
---
**📊 מבוסס אחוז**
ה-pill עוברת ל-DUE כשאחוז המשימות שהושלמו חוצה את הסף.

**טוב ל:**
- 25% מהעבודה
- 50% מהעבודה
- 75% מהעבודה
:::

## השלבים

1. **כנסו ללשונית כספים** בדף ההיתר (אם אינכם רואים את הלשונית — אתם לא {role:admin}).
2. לחצו **"+ הוסף אבן דרך"**.
3. מלאו את השדות:

| שדה | דוגמה |
|---|---|
| שם | `מקדמה — חתימת חוזה` |
| סכום | `50,000` ₪ |
| סוג טריגר | **משימה** או **אחוז** |
| משימה / אחוז | בחירה מתאימה |
| תאריך יעד (אופציונלי) | `30/07/2026` |

4. שמרו.

:::frame title="דיאלוג + אבן דרך"
**+ אבן דרך חדשה**

שם: `מקדמה — חתימת חוזה`
סכום: `50,000` ₪

סוג טריגר:  [ **משימה** | אחוז ]

**משימה:** [▾ [טפסים] חתימת חוזה]

תאריך יעד: `30/07/2026` (אופציונלי)

[ ביטול ]   [ צור ]
:::

## מחזור החיים — 3 סטטוסים

```
PENDING  →  DUE  →  PAID
   ↑        ↑        ↑
ממתין    מועד    סומן ידנית
        הגיע      ע"י אדמין
```

- **PENDING ← → DUE** הפיכות אוטומטית (לפי טריגר).
- **PAID** — רק ידנית. גם אם הסטטוס "נסוג" (משימה בוטלה, אחוז ירד) — `PAID` נשאר.

## איך מסמנים "שולם"

ב-`/finances` או בלשונית הכספים של ההיתר:

1. מצאו את האבן.
2. לחצו **"סמן כשולם"**.
3. הסטטוס משתנה ל-`PAID`, `paidAt` מתועד.

## טיפים

:::tip
**הגדירו אבני דרך מראש**, ברגע יצירת ההיתר. אל תחכו שהמשימה תושלם.
:::

:::tip
**אבן אחוז נוחה כשאין משימה אחת מובהקת.** לדוגמה "50% מהעבודה" כאן `triggerPercentage: 50` ימיר את הסטטוס אוטומטית כשתחצו 50% משימות הושלמו.
:::

:::warning
**אבני דרך באחוזים מתהפכות PENDING ↔ DUE.** אם משימה COMPLETED בוטלה, אחוז ההיתר ירד, ואבן הדרך תחזור ל-PENDING. אבל **PAID לא חוזר** — הוא נשאר.
:::
