# איך משנים הרבה משימות במכה אחת

מדריך כיס — פעולות bulk לחיסכון זמן.

:::info
**מי יכול?** {role:admin} ו-{role:employee}. מחיקה גורפת — רק {role:admin}.
**איפה?** בלשונית **משימות** של ההיתר, או ב-`/tasks` הגלובלי.
:::

## השלבים

1. **מסננים** קודם את הטבלה לסט הרצוי (לדוגמה: כל המשימות של אחראי X באיחור).
2. **מסמנים** את הצ׳ק-בוקסים מימין:
   - **לכל שורה** — צ׳ק-בוקס נפרד
   - **לכל הסט** — צ׳ק-בוקס "סמן הכל" בראש
3. בראש הטבלה צץ **סרגל פעולות צף** עם 3 פעולות.

:::frame title="סרגל פעולות צף — 12 משימות נבחרו"
**12 משימות נבחרו**

[ 👤 שנה אחראי ▾ ]   [ 🔄 שנה סטטוס ▾ ]   [ 🗑 מחק ] {role:admin}

[ × נקה בחירה ]
:::

## 3 הפעולות

### 1. שנה אחראי

dropdown של כל המשתמשים הפעילים. בחירה → כל ה-12 המשימות יקבלו את האחראי החדש. מתועד ב-AuditLog פר משימה.

### 2. שנה סטטוס

dropdown של כל הסטטוסים. בחירה → שינוי לכולן.

:::warning
**שינוי ל-COMPLETED בגורף** מפעיל `recalcPermitProgress` לכל המשימות. אבני דרך באחוזים יחושבו מחדש בסוף הפעולה.
:::

### 3. מחיקה {role:admin}

מחיקה רכה (soft-delete). הרשומות יעברו ל-`/settings/recycle-bin` ויהיו ניתנות לשחזור.

:::danger
**חסום אם משימה מעוגנת ל-Milestone לא משולם.** המערכת תציג רשימת המשימות החסומות ותבטל את הפעולה — מונע מחיקת אבן דרך לחיוב בטעות.
:::

## סינון לפני בחירה — כלל אצבע

אל תסמנו 100 משימות באופן ידני. במקום:

1. סננו לתת-סט (לדוגמה ב-`/tasks`: סטטוס=`AWAITING_AUTHORITY` + קטגוריה=`הג"א`).
2. לחצו "סמן הכל" — בודקים שה-N נראה הגיוני.
3. הפעילו את פעולת ה-bulk.

זה גם משאיר עקבות ברורים ב-AuditLog (אותו טקסט סינון = אותו רשם).

## טיפים

:::tip
**שינוי אחראי בעת חופשה** — בעובד שיוצא לחופש, סננו `/tasks` לפי אחראי=הוא + סטטוס=פעיל, סמנו הכל, וחנו לעובד אחר. זה ייקח 30 שניות.
:::

:::tip
**שינוי קטגוריה הגשה** — אם רשות עברה תהליך, סננו לפי קטגוריה, סמנו הכל, ושנו סטטוס ל-`AWAITING_AUTHORITY`. בלי לעבור על אחת-אחת.
:::

:::warning
**בלי undo גורף.** אם שיניתם 50 משימות בטעות, השחזור הוא ידני (עוברים על AuditLog). חשבו לפני שלוחצים.
:::
