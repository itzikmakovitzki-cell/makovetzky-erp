# WhatsApp ב-ERP

המערכת מתחברת ל-WhatsApp דרך **Green API** (חשבון לקוח) לשלושה תפקידים:

1. **קליטת מסמכים** מהשטח (תיבת WhatsApp)
2. **קבוצות פרויקט** — שיוך קבוצה ל-MasterDeal, קליטת הודעות לציר הזמן
3. **שליחה יוצאת ידנית** — תזכורות, איפוס סיסמה, עדכון ללקוח

:::danger
**חוק ברזל:** אין שום cron / scheduler / queue ששולח אוטומטית. כל שליחה היא תוצאה של **לחיצה ידנית של {role:admin}** בתוך המערכת. בלקוחות רגישים זה קריטי.
:::

## חלק א׳: תיבת WhatsApp — `/inbox`

### איך מסמכים נכנסים

הוובהוק ב-`/api/whatsapp/webhook` (Meta / Green API) או הוובהוק הגנרי ב-`/api/webhooks/whatsapp` מקבל הודעות נכנסות. כל מסמך נשמר כ-`PendingDocument` עם:

- שם קובץ + URL ב-Supabase Storage
- מקור (`WHATSAPP` / `EMAIL` / ...)
- פרטי שולח (שם, טלפון)
- ההודעה הגולמית
- סטטוס (`PENDING` / `ASSIGNED` / `REJECTED`)

ה-PendingDocument **לא משויך עדיין להיתר** — זאת ה-Triage שלכם.

לכל מסמך שתי פעולות:

1. **📁 שייך** — פותח דיאלוג עם:
   - פרויקט (MasterDeal)
   - היתר (מסונן אוטומטית לפי הפרויקט)
   - משימה או בניין (אופציונלי)
   - אישור ע"י {role:admin} (checkbox)

   לחיצה יוצרת `Document` רשמי תחת ההיתר ומעבירה את ה-`PendingDocument` ל-`ASSIGNED`.

2. **✗ דחה** — דורש סיבה חופשית. ה-`PendingDocument` עובר ל-`REJECTED` עם הסיבה.

:::tip
**שגרה יומית:** היכנסו ל-`/inbox` פעם בבוקר ופעם אחר הצהריים, ועברו על כל המסמכים. ככל שמטפלים בשטף, אין pileup ולא מאבדים מסמכים.
:::

## חלק ב׳: קבוצות פרויקט (WhatsApp Groups)

הרעיון: לחבר קבוצת WhatsApp של פרויקט ל-MasterDeal במערכת, כך שכל הודעה בקבוצה שתייגה את הבוט (`@mention`) או ציטטה הודעה שלו תיכנס לציר הזמן של הפרויקט.

### זרימת חיבור — 4 שלבים

1. **הוסיפו את הבוט** (Green API instance) לקבוצת WhatsApp של הפרויקט.
2. **ההודעה הראשונה** שמתקבלת מהקבוצה יוצרת רשומת **Orphan** — קיים `groupChatId` אבל אין `masterDealId`.
3. ב-`/inbox`, מעל תיבת המסמכים, יש פאנל **"קבוצות לא משויכות"** עם הקבוצה החדשה.
4. {role:admin} לוחץ **"שייך לפרויקט"** → בוחר MasterDeal → הקבוצה מקושרת.

### קליטת הודעות

הודעות שהבוט נתייג בהן (`@mention`) או הודעות שמצטטות את הבוט נשמרות כ-`PendingDocument` עם `groupChatId`, `authorName`, `authorPhone`, ואופציונלית `suggestedTaskName`.

### ציר זמן בפרויקט

ב-`/projects/[id]/whatsapp` יש tab ייעודי שמציג את כל ההודעות מהקבוצה שנקלטו (chronological, עם author, תצוגה מקדימה, drill-down).

## חלק ג׳: שליחה יוצאת ידנית

### תזכורת למבצע משימה

על כל משימה (בכל מסך שמראה את האחראי) יש כפתור 📱 ירוק. לחיצה:

1. **מעתיקה ל-clipboard** הודעה מוכנה:
   ```
   היי [שם], תזכורת לגבי משימה:
   [שם משימה] בפרויקט [שם פרויקט]
   אפשר סטטוס?
   ```
2. **פותחת `wa.me/[phone]?text=...`** בכרטיסייה חדשה (רק אם לאחראי יש מספר טלפון).

### שליחה ללקוח — דרך פרופיל הלקוח

ב-`/clients/[id]` יש פאנל WhatsApp:

:::frame title="פאנל WhatsApp בפרופיל לקוח"
**💬 WhatsApp**

מצב התקשרות: [ 🔕 כבוי | **✋ ידני בלבד** ]

──────────────────────────────────────────
☎ טלפון: 052-1234567

[ 📤 שלח עדכון ]
:::

לחיצה על "שלח עדכון" פותחת דיאלוג עם textarea ניתן לעריכה. כפתור השליחה משתנה לפי הטרנספורט:

:::cols
**Green API מוגדר**
`📤 שלח עכשיו`

דורש `window.confirm` נוסף ("לשלוח את ההודעה הזו ל-X?"). שליחה ישירה דרך השרת.
---
**Green API לא מוגדר**
`פתח WhatsApp`

נופל ל-wa.me — האדמין לוחץ שלח ב-WhatsApp עצמו.
:::

:::danger
**אם מצב התקשרות = "כבוי"** — ה-UI לשליחה **נמחק מה-DOM** (לא רק מנוטרל). אי אפשר להפעיל מיסקליק לסכואד. זה במכוון, ללקוחות רגישים.
:::

### Audit Log לכל שליחה

כל שליחה מתועדת ב-AuditLog עם אחת מ-3 פעולות:

| פעולה | מתי |
|---|---|
| `whatsapp_sent` | Green API הצליח, כולל `idMessage` |
| `whatsapp_send_failed` | Green API נפל, מצוטט שגיאה ונפילה ל-wa.me |
| `whatsapp_opened` | wa.me נפתח (fallback או לא מוגדר) |

### תצורת Green API

שני env vars שצריכים להיות מוגדרים ב-Vercel לשליחה ישירה:

```
GREEN_API_ID_INSTANCE
GREEN_API_TOKEN_INSTANCE
```

ניתן להוציא מ-[console.green-api.com](https://console.green-api.com/). אם לא מוגדרים, המערכת חוזרת אוטומטית ל-wa.me.

## חלק ד׳: איפוס סיסמה דרך WhatsApp

המשתמש שכח סיסמה? `/forgot-password` ציבורי לחלוטין:

1. ב-`/forgot-password` המשתמש מכניס את **מספר הטלפון** שלו (לא אימייל).
2. השרת מחפש משתמש עם הטלפון הזה ב-DB.
3. **אם נמצא** — נשלח `PasswordResetToken` (32 תווים, תוקף שעה, חד-פעמי) דרך Green API:
   ```
   קישור לאיפוס סיסמה:
   https://crm.makomedia.co.il/reset-password/[token]
   ```
4. **אם לא נמצא** — מוצגת **אותה תגובה** ("בדוק את ה-WhatsApp שלך"). הגנה מ-account enumeration.

:::warning
**דרישות סביבה:** `SYSTEM_WHATSAPP_PHONE`, `SYSTEM_WHATSAPP_NAME`, ו-**`NEXT_PUBLIC_APP_URL`** חייבים להיות מוגדרים ב-Vercel. ההודעה משתמשת ב-URL לבניית הקישור — בלעדיו נופל ל-`VERCEL_URL` עם דומיין `.vercel.app`.
:::

### דף איפוס `/reset-password/[token]`

בודק שהטוקן קיים, לא פג תוקף, לא בשימוש. אם תקין — שדות סיסמה חדשה + אישור → שמירה → `Token.usedAt` מסומן → המשתמש מועבר ל-`/login`.

## טיפים שיחסכו לכם כאב ראש

:::tip
**הקפידו שלכל משתמש יש טלפון** ב-`/settings/users`. בלעדיו, כפתורי WhatsApp לא יעבדו ועלולים להציג "טרם הוזן".
:::

:::tip
**לקוחות רגישים — סמנו OFF** בפרופיל הלקוח. ה-UI לשליחה לא יוצג בכלל, לא רק מנוטרל.
:::

:::warning
**Green API mass-send אסור.** המערכת תוכננה במכוון בלי scheduler. אל תוסיפו cron שעוקף את זה.
:::

:::tip
**בדקו את ה-AuditLog** אחרי שליחה — `whatsapp_sent` מאשר שעבר, `whatsapp_send_failed` מסביר למה נכשל.
:::

:::info
**קבוצות orphans** — אם הקבוצה כבר לא רלוונטית, אפשר לדחות אותה מ-/inbox, או להשאיר אם תתקבל הודעה חדשה בעתיד.
:::
