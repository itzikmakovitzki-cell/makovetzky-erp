# כספים — אבני דרך ועמלות

המודול הפיננסי נחלק לשני עולמות נפרדים:

1. **אבני דרך לחיוב** (Billing Milestones) — מה לקוחות חייבים לכם, מתי, ועל סמך מה.
2. **עמלות מספקים** (Supplier Commissions) — מה אתם מקבלים כעמלה על תיווך עבודה לספקים שלכם.

המודול **רק לעיני מנהל (ADMIN)**. עובד רגיל לא רואה את `/finances` בכלל בסיידבר.

> 🔒 **פרטיות מובנית:** מאז Block 23, אין מספרים פיננסיים על דפי הניווט הראשיים. במקום toggle "פרטיות" שהיה פעם, ה-`/projects` ו-`/permits` פשוט לא מציגים מחירים. הכל סגור מאחורי `/finances` או כפתור "ניהול פיננסי" (Sheet) בדף ההיתר/פרויקט. אפשר לפתוח לקוח לידכם בלי לחשוף סכומים.

## חלק א׳: אבני דרך לחיוב

### מבנה אבן דרך

כל אבן דרך מוגדרת ברמת **היתר** ויש לה:

| שדה | מה זה |
|---|---|
| `name` | שם אבן הדרך, לדוגמה "מקדמה — חתימת חוזה" |
| `amount` | סכום בש"ח |
| `triggerKind` | **משימה** או **אחוז** (XOR — חייב להיות אחד מהם) |
| `triggerTaskId` | אם מבוסס משימה — מזהה משימה |
| `triggerPercentage` | אם מבוסס אחוז — מספר 1–100 |
| `status` | `PENDING` → `DUE` → `PAID` |
| `dueDate` | תאריך יעד (אופציונלי) |

### שני סוגי טריגרים

**1. מבוסס משימה (Task-based)** — אבן הדרך עוברת ל-`DUE` כשהמשימה המעוגנת מסומנת `COMPLETED`. שימושי ל-milestones קלאסיים: "בקבלת אישור הג"א", "בקבלת היתר בנייה", "בקבלת טופס 4".

**2. מבוסס אחוז (Percentage-based)** — אבן הדרך עוברת ל-`DUE` כשאחוז המשימות שהושלמו בהיתר חוצה את הסף שהוגדר. שימושי ל-milestones הדרגתיים: "25% מהעבודה", "50%", "75%".

החישוב מבוצע בפעולת `recalcPermitProgress` שרצה בכל פעם שמשימה משנה סטטוס לקריטריון COMPLETED. ה-`Permit.progressPercent` מתעדכן, ואבני דרך באחוזים מתהפכות אוטומטית `PENDING ↔ DUE` (אבל לעולם לא הופכות לבד ל-`PAID` — זה דורש פעולה ידנית).

### יצירה ועריכה

בלשונית הכספים של ההיתר (פותחים מ-Sheet כפתור "ניהול פיננסי"). דיאלוג "+ הוסף אבן דרך" עם pill שמתחלף בין שני סוגי הטריגרים:

- בחירת **משימה** → רואים dropdown של משימות ההיתר.
- בחירת **אחוז** → רואים input של 1–100 + תצוגה חיה של מד התקדמות (אחוז מטרה + אחוז נוכחי + ספירת הושלם/סה"כ + פס שמתחלף לירוק כשמגיעים ל-`DUE`).

### תצוגה גלובלית — `/finances`

עמוד `/finances` מציג את **כל אבני הדרך** מכל ההיתרים. שלושה סיכומים בראש: **ממתין** / **מועד הגיע** / **שולם**. מסנן סטטוס נשמר ב-URL (אפשר לסמן כסימנייה).

עמודות הטבלה: לקוח, היתר, אבן דרך, משימה מפעילה, סכום, תאריך יעד, סטטוס. במובייל — כרטיסים.

### תהליך תשלום

1. אבן דרך מגיעה ל-`DUE` (אוטומטית כשהמשימה הושלמה / האחוז הושג).
2. אתם שולחים חשבונית ללקוח (מחוץ למערכת).
3. הלקוח משלם.
4. ב-`/finances` או בלשונית הכספים של ההיתר, לוחצים **"סמן כשולם"**. הסטטוס משתנה ל-`PAID`, `paidAt` מתועד.

## חלק ב׳: עמלות מספקים (Block 27)

המערכת מנהלת את הספקים שלכם (מודדים, אדריכלים, הג"א, וכו') ומחשבת אוטומטית כמה עמלה מגיעה לכם.

### הקמת ספק

ב-`/suppliers`, כפתור "ספק חדש". שדות:

- שם
- סוג (מודד / יועץ / קבלן / ...) עם autocomplete
- אתר, איש קשר, טלפון, אימייל
- **שירותים** — טקסט חופשי המתאר מה הספק עושה
- **עמלת ברירת מחדל** — `FIXED` (סכום ש"ח) או `PERCENT` (אחוז מסך המשימה)
- **תנאי תשלום ברירת מחדל** — טקסט חופשי ("שוטף+30", "מקדמה 50%")

### הקצאה של ספק למשימה

בעמוד הספק (`/suppliers?supplier=...`) או בכפתור "+ הקצה" בעמוד הספקים. הקצאה כוללת:

- משימה
- סכום (Base amount) — אם העמלה PERCENT, מחושבת על זה
- **עמלה ספציפית** (override) — אופציונלי. אם לא, יורש מהספק.
- **תנאי תשלום** — אופציונלי. אם לא, יורש.
- סטטוס (`OPEN` / `IN_PROGRESS` / `COMPLETED` / `CANCELLED`)
- `commissionPaidAt` — מתי שולמה לי העמלה. ניתן לסמן ב-toggle אחד.

הרזולוציה של הסכום היא **בזמן קריאה** — אם משנים את ברירת המחדל של הספק, ההקצאות שלא overridden מתעדכנות אוטומטית בלי backfill.

### דשבורד עמלות — `/finances/supplier-commissions`

זה ה"מה מגיע לי" שלכם:

3 קלפי סיכום:

- **הושלם בתקופה** — הקצאות שעברו ל-COMPLETED בתוך החלון. הרגע שבו "הרווחת" את העמלה.
- **שולם בתקופה** — `commissionPaidAt` בתוך החלון. מנותק מתי הושלמה העבודה — תזמון תשלום שייך לחלון אחר.
- **יתרה — מה שמגיע לי** — הקצאות שהושלמו עד סוף החלון, `commissionPaidAt = null`. **מצטבר**, לא מוגבל לתחילת החלון. זה המספר האמיתי של "כמה חייבים לי".

מסנן תקופה: **החודש** / **חודש שעבר** / **מתחילת השנה** / **טווח מותאם** (תאריך מ-עד). נשמר ב-URL.

טבלה לפי ספק, מסודרת לפי יתרה יורדת. שורת סה"כ למטה.

לחיצה על שם ספק → `/suppliers?supplier=X&all=true` — רואים את כל ההקצאות שלו.

### ייצוא Excel

כפתור הייצוא בעמוד עמלות מספקים מוריד `.xlsx` ברמת הקצאה (לא מקובצות) עם תיוג סטטוס תשלום (`שולם` / `מחכה לתשלום` / `פעיל`). שורת סה"כ עם 3 המספרים.

## טיפים פיננסיים

- **הגדירו עמלת ברירת מחדל לכל ספק** — וברוב המקרים תנו לאבני דרך ספציפיות לרשת. רק כשיש סטייה, השתמשו ב-override.
- **סמנו `commissionPaidAt` ברגע שקיבלתם** — לא "אחרי החודש". זה מה שמייצר את היתרה.
- **שמרו ייצוא Excel בסוף חודש** — Snapshot של מה שהיה במצב הזה. שימושי לרואה החשבון.
- **אבני דרך באחוזים נוחות** — אבל זכרו: הן מתהפכות `PENDING ↔ DUE` אוטומטית. אם אחוז ההיתר ירד (משימה הופנתה חזרה מ-COMPLETED), אבן הדרך תחזור ל-`PENDING`. רק `PAID` לא הופך לבד.
