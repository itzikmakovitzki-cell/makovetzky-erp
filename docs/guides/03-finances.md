## חלק א׳: אבני דרך לחיוב

### מבנה אבן דרך

כל אבן דרך מוגדרת ברמת **היתר** ויש לה:

| שדה | תפקיד |
|---|---|
| `name` | שם, לדוגמה "מקדמה — חתימת חוזה" |
| `amount` | סכום בש"ח |
| `triggerKind` | **משימה** או **אחוז** (XOR — בדיוק אחד) |
| `triggerTaskId` | אם מבוסס משימה |
| `triggerPercentage` | אם מבוסס אחוז (1-100) |
| `status` | `PENDING` → `DUE` → `PAID` |
| `dueDate` | תאריך יעד (אופציונלי) |

### שני סוגי טריגרים

:::cols
**1. מבוסס משימה**
אבן הדרך עוברת ל-`DUE` כשהמשימה המעוגנת מסומנת `COMPLETED`.

שימושי ל-milestones קלאסיים:
- בקבלת אישור הג"א
- בקבלת היתר בנייה
- בקבלת טופס 4
---
**2. מבוסס אחוז**
אבן הדרך עוברת ל-`DUE` כשאחוז המשימות שהושלמו בהיתר חוצה את הסף.

שימושי ל-milestones הדרגתיים:
- 25% מהעבודה
- 50%
- 75%
:::

:::tip
**איך נמדד האחוז?** הפעולה `recalcPermitProgress` רצה בכל פעם שמשימה משנה סטטוס. `Permit.progressPercent` מתעדכן, ואבני דרך באחוזים מתהפכות אוטומטית `PENDING ↔ DUE`. **שימו לב:** הן לא הופכות לבד ל-`PAID` — זה דורש פעולה ידנית.
:::

### יצירה ועריכה

בלשונית הכספים של ההיתר (פותחים מ-Sheet כפתור "ניהול פיננסי"). דיאלוג **"+ הוסף אבן דרך"**:

:::frame title="דיאלוג יצירת אבן דרך"
**+ אבן דרך חדשה**

שם: `_____________________`
סכום: `[ 50,000 ] ₪`

סוג טריגר:  [ **משימה** | אחוז ]

⤷ **משימה:**  [▾ בחר משימה...]

⤷ **אחוז:**  [ 50 ]%
&nbsp;&nbsp;&nbsp;📊 מצב נוכחי: 32% (8/25 הושלמו)
&nbsp;&nbsp;&nbsp;━━━━━━━━━░░░░░░░░░░ → DUE כשמגיע ל-50%

תאריך יעד: `[ 30/07/2026 ]` (אופציונלי)

[ ביטול ]  [ צור ]
:::

### תצוגה גלובלית — `/finances`

עמוד `/finances` מציג את **כל אבני הדרך מכל ההיתרים**, עם 3 קלפי סיכום בראש:

:::frame title="/finances — דשבורד אבני דרך"
**ממתין**            **מועד הגיע**            **שולם**
180,000 ₪           75,000 ₪              420,000 ₪
(12 רשומות)         (3 רשומות)             (28 רשומות)

🔍 סינון: [ ▾ סטטוס: הכל ] [ ▾ לקוח ] [ ▾ תקופה ]

| לקוח | היתר | אבן דרך | טריגר | סכום | יעד | סטטוס |
|---|---|---|---|---|---|---|
| יעקב כהן | מגדל המשי | מקדמה | חוזה ✓ | 50K | 01/05 | 🟢 שולם |
| ש. בנייה | פרויקט הרצליה | 50% עבודה | אחוז | 35K | 15/07 | 🟠 מועד |
| יעקב כהן | מגדל המשי | אישור הג"א | משימה | 25K | — | ⚪ ממתין |
:::

מסנן סטטוס נשמר ב-URL (אפשר לסמן כסימנייה).

### תהליך תשלום — 4 שלבים

1. **אבן דרך מגיעה ל-`DUE`** אוטומטית כשהמשימה הושלמה / האחוז הושג.
2. **שולחים חשבונית ללקוח** (מחוץ למערכת).
3. **הלקוח משלם**.
4. **סימון "שולם"** ב-`/finances` או בלשונית הכספים של ההיתר. הסטטוס משתנה ל-`PAID`, `paidAt` מתועד.

## חלק ב׳: עמלות מספקים

המערכת מנהלת את הספקים שלכם (מודדים, אדריכלים, הג"א, וכו') ומחשבת אוטומטית כמה עמלה מגיעה לכם.

### הקמת ספק

ב-`/suppliers`, כפתור "ספק חדש":

| שדה | הערה |
|---|---|
| שם | חובה |
| סוג | מודד / יועץ / קבלן / ... (עם autocomplete) |
| אתר, איש קשר, טלפון, אימייל | פרטי קשר |
| שירותים | טקסט חופשי המתאר מה הספק עושה |
| **עמלת ברירת מחדל** | `FIXED` (סכום ₪) או `PERCENT` (אחוז מסך המשימה) |
| **תנאי תשלום ברירת מחדל** | "שוטף+30", "מקדמה 50%" |

### הקצאה של ספק למשימה

בעמוד הספק או בכפתור "+ הקצה" בעמוד הספקים:

:::frame title="הקצאת ספק למשימה"
**הקצאת ספק:** מודדי השרון

**משימה:** [▾ [טפסים] מדידת מצב קיים — מגדל המשי]

**סכום בסיס:** `[ 8,000 ]` ₪
&nbsp;&nbsp;&nbsp;⤷ העמלה תחושב: 5% מ-8,000 = **400 ₪**

**עמלה ספציפית (override):** [ ___ ] ₪ או [ ___ ]%
&nbsp;&nbsp;&nbsp;*ריק = יורש מברירת המחדל של הספק (5%)*

**תנאי תשלום (override):** `_____________________`

**סטטוס:** [▾ OPEN]
**שולמה לי העמלה?** ☐  `_________` (תאריך)
:::

:::tip
**רזולוציה בזמן קריאה.** אם תשנו את ברירת המחדל של הספק, ההקצאות שלא overridden יתעדכנו אוטומטית בלי backfill. ברירת מחדל = מקור אמת חי.
:::

### דשבורד עמלות — `/finances/supplier-commissions`

זה ה"מה מגיע לי" שלכם. שלושה קלפים בראש:

:::frame title="/finances/supplier-commissions"
**הושלם בתקופה**       **שולם בתקופה**       **יתרה — מגיע לי**
12,500 ₪              8,000 ₪              **18,750 ₪**
(15 הקצאות)           (10 הקצאות)          (23 הקצאות, מצטבר)

🔍 תקופה: [ **החודש** | חודש שעבר | מתחילת השנה | טווח מותאם ]

| ספק | הקצאות פעילות | מגיע | שולם החודש |
|---|---|---|---|
| מודדי השרון | 4 | 5,200 ₪ | 1,200 ₪ |
| הג"א תעשיות | 2 | 7,500 ₪ | 0 ₪ |
| אדריכל לוין | 8 | 6,050 ₪ | 6,800 ₪ |
:::

### מה ההבדל בין 3 הקלפים?

| קלף | מה זה אומר |
|---|---|
| **הושלם בתקופה** | הקצאות שעברו ל-COMPLETED בתוך החלון. הרגע שבו "הרווחת" |
| **שולם בתקופה** | `commissionPaidAt` בתוך החלון. תזמון תשלום, לא של עבודה |
| **יתרה — מגיע לי** | הקצאות שהושלמו עד **סוף החלון**, `commissionPaidAt = null`. **מצטבר**, לא חתוך — זה המספר האמיתי של "כמה חייבים לי" |

טבלה מסודרת לפי **יתרה יורדת**. שורת סה"כ למטה. לחיצה על שם ספק → `/suppliers?supplier=X&all=true`.

### ייצוא Excel

כפתור הייצוא מוריד `.xlsx` ברמת הקצאה (לא מקובץ) עם תיוג סטטוס תשלום (`שולם` / `מחכה לתשלום` / `פעיל`). שורת סה"כ עם 3 המספרים.

## טיפים פיננסיים

:::tip
**הגדירו עמלת ברירת מחדל לכל ספק.** ברוב המקרים תנו לה לרוץ — רק כשיש סטייה ספציפית, השתמשו ב-override.
:::

:::tip
**סמנו `commissionPaidAt` ברגע שקיבלתם** — לא "אחרי החודש". זה מה שמייצר את היתרה האמיתית.
:::

:::tip
**שמרו ייצוא Excel בסוף חודש** — snapshot של מה שהיה במצב הזה. שימושי לרואה החשבון.
:::

:::warning
**אבני דרך באחוזים — שימו לב:** הן מתהפכות `PENDING ↔ DUE` אוטומטית. אם אחוז ההיתר ירד (משימה הופנתה חזרה מ-COMPLETED), אבן הדרך תחזור ל-`PENDING`. **רק `PAID` לא הופך לבד** — היא נשארת `PAID` עד שתבטלו ידנית.
:::
